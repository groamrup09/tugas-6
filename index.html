Multiple Inheritance dengan Mixin

        * Dart menggunakan keyword Mixins untuk menggunakan beberapa class yang
          berbeda dalam satu class, tanpa perlu membuat inheritance yang rumit.
        * beberapa bahasa yang mendukung multiple inheritance adalah C++, Python,
          Scala, Perl, DLL sedangkan Java, C#, dan Kotlin mendukung Single inheritance
        * Multiple inheritance (pewarisan ganda/banyak) digunakan untuk
          menyelesaikan masalah di mana sebuah objek (subclass) memiliki perilaku atau
          properti dari dua atau lebih induk yang berbeda (superclass) secara bersamaan,
          menggabungkan fungsionalitas tanpa redundansi kode. Contohnya adalah
          pemodelan objek kompleks seperti "Bebek" (Mewarisi bisa berenang dan bisa
          terbang) atau "Smartphone" (Mewarisi Kamera dan Telepon).

// Mixin dengan Properti (Opsional)
mixin CanFly {
  bool _isFlying = false;
  bool get isFlying => _isFlying;
  
  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }
  
  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    } else {
      print('Already on the ground!');
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false;
  bool get isSwimming => _isSwimming;
  
  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    } else {
      print('Already swimming!');
    }
  }
  
  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    } else {
      print('Already stopped swimming!');
    }
  }
}

Pengantar Class Modifiers

Apa itu? Keyword untuk mengontrol bagaimana class dapat digunakan
Tujuan:
i. Kontrol inheritance/implementasi
ii. Jaminan keamanan tipe
iii. Desain API yang lebih ekspresif
Macam-macam: 
sealed , 
base , 
final , 
interface (Dart 3.0+)

Sealed Class

        Tidak bisa di-extend atau di-implement dari luar library
        Membatasi hierarki tipe yang diketahui saat compile-time
        Perfect untuk pattern matching (switch exhaustive)

File 1: libsealed.dart

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

class Loading extends ApiResponse {
  final String status;
  Loading(this.status);
}

Keuntungan Sealed Class

1. Exhaustive checking: Compiler tahu semua kemungkinan subclass
2. Pattern matching: Switch case yang aman
3. Documentation: Jelas hierarki terbatas

Praktik 1 – Sealed Class dengan Pattern Matching

// Nama   : Azriel Miracle Yuandre
// Nim    : 251410024
// Kelas  : Si2A 

sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print('Loading...');
      break;
    case Loaded(:final data):
      print('Data: $data');
      break;
    case Failed(:final error):
      print('Error: $error');
      break;
  }
}

void main() {
  handleStatus(const Loading());
  handleStatus(const Loaded("Data berhasil"));
  handleStatus(const Failed("Terjadi error"));
}

// Nama   : Azriel Miracle Yuandre

sealed class AppState {}

class LoadingState extends AppState {}

class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

// Handler
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
      break;
    case SuccessState(:final items):
      showItems(items);
      break;
    case ErrorState(:final message):
      showError(message);
      break;
  }
}

void showLoading() {
  print('\nLoading...');
}

void showItems(List items) {
  print('\nSuccess! Items:');
  for (int i = 0; i < items.length; i++) {
    print('   ${i + 1}. ${items[i]}');
  }
}

void showError(String message) {
  print('\nError: $message');
}

void main() {
  updateUI(LoadingState());
  updateUI(SuccessState(["Apple", "Banana", "Orange"]));
  updateUI(ErrorState("Gagal mengambil data"));
}

Ringkasan & Best Practices
1. Sealed: Hierarki tertutup, pattern matching aman
2. Base: Hanya extend (no implement) dari luar
3. Final: No extend/implement dari luar
Best Practices:
1. Gunakan 
sealed untuk state/result yang terbatas
2. Gunakan 
base untuk framework/base class
3. Gunakan 
final untuk utility class
